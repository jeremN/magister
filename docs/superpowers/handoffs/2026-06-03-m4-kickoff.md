## 2. One M3 architectural decision you must know (it's blessed, keep it)

**`core.Store` is no longer "frozen" — it gained `AppendEvents(ctx, id, []event.Event) error`** (+ `Mem`/`SQLite` impls). The SSE hub sources content from `store.EventsSince` (real seqs), but run-level events (`run.started`/`run.done`) were only bus-published, never persisted — so SSE couldn't replay them or terminate. The engine's `runDAG` now persists run-level events via `AppendEvents` **before** `Bus.Publish` (persist-then-publish; same monotonic seq source as `SaveStepTransition`). This was reviewed (opus) and **explicitly approved by the user**. Do not revert it. Build on it.

## 4. M3 follow-ups (deferred, none block M4 — fold the cheap ones in when you touch that code)

From the final holistic review + execution. None are correctness bugs in the happy path.

- **Resume staleness → spurious 409 on approve (product wrinkle).** On resume, a non-succeeded step shows its *pre-crash* status (e.g. `awaiting_gate`) until the engine re-runs it and re-registers the gate. An approve landing in that window gets `Resolve()==false` → **409**. Correct client behavior is **retry** (the e2e `approveStep` helper does this; `cm approve` should too — consider a retry-on-409-while-awaiting loop in `cm`). A cleaner server-side fix to weigh in M4: on resume, reset non-terminal steps to `pending` (so observers don't see a stale actionable status), OR make the ApprovalRegistry buffer a "resolve-before-await" decision. This is the most important follow-up.
- **`Supervisor.ResumeAll` fail-fast:** returns on the *first* parse/validate error, so one corrupt persisted `FlowYAML` row strands every *later* run in the slice. Non-fatal to the daemon (logged), but consider log-and-continue per row.
- **Dead `Server.BearerToken` / `Server.ShutdownTimeout` fields** (`internal/api/handlers.go`): set by the daemon but never read (auth token flows via `Router(token)`, shutdown via `cfg`). **Latent auth footgun:** a future caller that sets `srv.BearerToken` then calls `srv.Router("")` silently disables auth. Either drop the fields or make `Router()` default to `s.BearerToken`. Cheap; do it when you touch the api package.
- **`sse.go` dereferences `s.Log` directly** (not nil-safe like the engine's `logger()` helper). Safe today (daemon + all tests wire a real logger); add an `api.Server` `logger()` helper if you harden it.
- **Canceled steps persist as `failed`/`awaiting_gate`, not `canceled`** (spec §4 says steps → `canceled`). Pre-existing engine behavior (M3 didn't touch the cancel branch); cosmetic — run-level status + `run.done` are correct.
- **`magisterd` graceful-shutdown can take up to 2×`ShutdownTimeout`** when an SSE client stays connected (SSE ignores `http.Server.Shutdown`; only exits on `r.Context().Done()`). And a second SIGINT/SIGTERM during drain is swallowed (no force-quit). Both are deliberate `WriteTimeout:0` tradeoffs; revisit with a shared shutdown budget or a base-context cancel that closes SSE streams.

## 5. Conventions & gotchas (carried from M0–M3, strict)

- **Commits (user CLAUDE.md):** single conventional-commit subject line, **no body**, **never** a `Co-Authored-By` trailer, **never** `--no-verify`. Git identity isn't global — commit with:
  ```bash
  git -c user.name="Jérémie Nehlil" -c user.email="jeremie.nehlil.freelance@proton.me" commit -m "<subject>"
  ```
- **Dep pins (go-1.22 must not move):** `modernc.org/sqlite v1.36.1`, `pressly/goose/v3 v3.24.1` (next patch needs go 1.23). `github.com/oklog/ulid/v2 v2.1.1` is fine. After any `go get`, confirm `grep '^go ' go.mod` is still `go 1.22`. `sqlc` is dev-time; generated code is checked in.
- **RTK hook** reformats `go`/`git` output (e.g. "Go test: N passed"). When you need raw PASS/FAIL lines (esp. per-iteration flake hunts), use `rtk proxy go test ...`.
- **Semgrep hook** runs on edits. It flags `fmt.Fprintf(w, ...)`/`w.Write` to an `http.ResponseWriter` as CWE-79 (accepted false positives on the loopback SSE/JSON surfaces — documented in-code). It *warns* on most edits but *blocked* a test helper's `w.Write` once; the workaround was routing the write through an `io.Writer` parameter. Don't switch SSE to `html/template`. `crypto/subtle.ConstantTimeCompare` (bearer auth) is correct if flagged.
- **Test flakiness discipline (learned in M3 Task 15):** concurrency/lifecycle tests MUST be stress-verified — `GOMAXPROCS=8 go test -race -count=N` exposes scheduling races a single `-count=5` run misses (the e2e kill/resume test flaked at GOMAXPROCS=8 and was rewritten to a *deterministic* crash: submit in-process, poll the store until `awaiting_gate`, then close the store **without** graceful shutdown — no cancel, no `RunCanceled` write race; run stays `running` for `LoadIncompleteRuns`).
- **Engine invariants (don't break):** lives in `internal/engine` (not `core` — import cycle); persist-then-publish (durable write, then publish; on store error don't publish the original); SQLite single-writer (`SetMaxOpenConns(1)`) + reader pool; deadlock-freedom (a step waits on dep channels before acquiring any token); `Engine.Log` (nil = discard) wired to a real handler by the daemon.
- **Output style** is learning/explanatory — include brief `★ Insight` notes when teaching something codebase-specific.
