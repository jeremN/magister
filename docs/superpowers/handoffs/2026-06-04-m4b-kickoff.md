## 4. Key facts the plan relies on (so you don't re-derive them)

- **`claude` CLI headless (researched, current):** `claude -p "<prompt>"` (prompt is the positional arg after `-p`); `--model opus|sonnet` (aliases); `--output-format json` â‡’ a **single result object** on stdout (we chose `json`, NOT `stream-json`, for simplicity â€” stream-json + live SSE is deferred); `--permission-mode acceptEdits` lets it edit files non-interactively (chosen over `--dangerously-skip-permissions`); operates on cwd (`cmd.Dir`); auth via **`ANTHROPIC_API_KEY`** in env. Result fields used: `total_cost_usd` (float), `result` (string summary), `is_error` (bool) + `subtype` (`"success"` vs `error_*`). Unknown JSON fields are ignored.
- **Secrets:** `CLIAgent.Env == nil â‡’ os.Environ()` passes the key to the subprocess; secrets never touch flow YAML/args/logs. `exec.Command` with an arg slice (no shell); annotate `#nosec G204` with the trust-boundary rationale (precedent: `gate/verifier.go`, `workspace/gitmanager.go`).
- **Artifacts:** `git status --porcelain` in the workdir â†’ changed files become `core.Artifact{StepID, Path:<abs>}`. Works because Slice C's `GitManager` makes every step's workdir a git tree. Discovery failure is **non-fatal** (logged via a nil-safe `CLIAgent.Log`, empty artifacts).
- **Slice C context you build on:** the daemon wires `workspace.GitManager` (isolated step â†’ real `git worktree` off a scratch per-run repo; shared step â†’ base tree; worktrees torn down at run end). B1's agents run inside those worktrees, greenfield (empty base commit, so everything the agent writes is its output).
- **Failure semantics:** a failed agent (missing binary / non-zero exit / parse error / `is_error`) is an **executor error** â†’ retryable under Slice A's unified attempt budget, then abort/escalate per `on_fail`. The per-step timeout (Slice A) kills a hung agent via `ctx`.
