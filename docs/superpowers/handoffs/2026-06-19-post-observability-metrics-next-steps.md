## Known caveat (NOT a bug)
`magister_build_info{version=…}` showed the worktree's **base** commit (`14b323c`), not HEAD (`ebc58dd`), during the live proof. Root cause: Go's `-buildvcs` stamps the wrong revision when building inside a **linked git worktree** (`go version -m` confirmed `vcs.revision=14b323c`, `vcs.modified=false`). `BuildVersion()` is a faithful pass-through of whatever Go stamps — verified correct. A normal/CI build of merged `main` will stamp the right commit. No code change needed.
