## What shipped

`cm ship <run> --head-repo <fork>` now delivers a succeeded external-repo run to a **fork** in one command: it pushes `magister/<run>` to the fork **and** opens a **cross-fork** PR into the upstream (head `forkowner:branch`, base upstream/origin), idempotently. This completes the fork-delivery story (`cm push --remote <fork>` + `cm pr --head-repo <fork>` were already fork-capable; ship now collapses them). Spec `docs/superpowers/specs/2026-06-22-cm-ship-head-repo-design.md`, plan `docs/superpowers/plans/2026-06-22-cm-ship-head-repo.md`.

**Mechanism (minimal — threads one field through 4 layers).** `ShipOpts` gained `HeadRepo string`. `Supervisor.Ship` makes two changes: (1) **push routes to the fork** — `pushRemote := opts.Remote; if opts.HeadRepo != "" { pushRemote = opts.HeadRepo }` → `Push(PushOpts{Remote: pushRemote, …})`; (2) **PR stays cross-fork** — `prCore(PROpts{Remote: opts.Remote, …, HeadRepo: opts.HeadRepo})`. The base stays `opts.Remote`/origin (upstream); the head becomes `forkOwner:branch` inside the already-merged `prCore`. **No change to `internal/host`, push (`internal/workspace`), or `prCore` logic** — Ship only threads the field.

## Key properties (Opus-verified)

- **Same-repo path byte-for-byte unchanged.** `HeadRepo==""` ⇒ `pushRemote==opts.Remote` and the `prCore` call adds only an empty `HeadRepo` (which `prCore` skips). The pre-existing `TestShipPushFailsSkipsPR`/`TestShipPushesThenPropagatesPRError` regressions are unaffected.
- **Routing split correct.** Push → `--head-repo` (fork); PR base → `--remote`/origin (upstream); the two never both feed the push. In fork mode `--remote` overrides only the PR base (its `cm pr` meaning) — the chosen semantic.
- **Idempotency is free.** `prCore` uses the fixed `ExistingOpenPR`; a fork-ship re-run re-pushes the same SHA (git "up-to-date", no `--force` needed) and reports `PRExisted=true` → prints `exists <url>`, no duplicate PR.
- **The fully-happy fork path is not offline-unit-testable** (one URL can't be both an offline-pushable bare repo and a github-parseable head owner) — covered by the live smoke. The offline tests use a **`file://` URL** for the fork (the supported passthrough form) to prove push-routing + threading via the failure paths.
